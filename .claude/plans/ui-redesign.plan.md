# UI 大改方案 — 浅色清雅(墨蓝 · 书卷气) + 视觉/布局双优化

## 一、目标与约束

**目标**:把当前「灰阶极简」界面升级为「浅色清雅、优雅大气现代」,强化交互体验。
**风格定调**(已确认):暖白底 + 墨蓝点缀 + 书卷气克制。
**改动幅度**(已确认):视觉 + 布局优化(CSS 重写为主,Python 侧小幅结构调整)。

**关键约束**(已勘查):
- 样式集中在 `src/web.py` 的 `CUSTOM_CSS` 常量,`app.py` 和 `main.py` 两入口都用 `launch(theme=Soft, css=CUSTOM_CSS)`,改常量即全局生效。
- 基于 Gradio Blocks,三大悬浮区块:`#input-row`(输入)、`#agent-mode-row`(深度模式)、`#examples-wrap`(空状态),都 `position:absolute` 悬浮在 `#chat-shell` 内。
- `sanitize_html=False`,页脚/轨迹/引用卡片是手写 HTML 注入,改它们的样式即改 CSS class。
- 不动任何业务逻辑(chat_stream / 流式 / 反馈 / 锚点跳转),只动呈现层。

## 二、设计令牌(新配色体系)

替换现有 `:root` 变量:

```
--bg:          #f5f3ef   暖米白(替代冷灰 #f7f7f8)
--bg-grad:     顶部 #f7f5f1 → 底部 #efece6 极淡竖向渐变,增加层次
--surface:     #ffffff   卡片纯白
--surface-alt: #faf9f7   次级面(引用块底)
--border:      #e7e3da   暖调弱描边(替代冷灰 #ebebed)
--text:        #1c2733   近墨黑(带蓝调)
--muted:       #6b7280   中性灰
--accent:      #1e3a5f   墨蓝(主点缀:用户气泡、按钮、链接、强调)
--accent-soft: #2c5282   墨蓝亮一档(hover)
--accent-wash: #eef2f7   墨蓝极淡(选中/锚点高亮底)
--gold:        #b08d57   暖金(极克制,仅图标/分隔线点缀,书卷气)
```

字体:正文保持系统字体栈;**标题/品牌**加一档衬线体(`"Noto Serif SC", "Songti SC", serif`)强化书卷气,仅用于 H1 和欢迎语。

## 三、分区改造清单

### 1. 顶栏 `#app-header`(视觉)
- H1 改墨蓝色 + 衬线体,⚖️ 图标换成墨蓝/金色调。
- 标题下加一条极细墨蓝→透明的渐变分隔线(替代当前无分隔)。
- 副标题字号/色阶微调,更精致。

### 2. 对话外壳 `#chat-shell` / `#chatbot`(视觉+布局)
- 圆角 20→18px,描边改暖调,阴影更柔(多层柔和阴影替代单层)。
- 背景:纯白 → 极淡暖白,与外层米白拉开层次。
- 消息间距 16→18px,呼吸感更足。

### 3. 消息气泡 `.message`(视觉,体验核心)
- **用户气泡**:深灰 #1f2937 → 墨蓝 #1e3a5f,圆角更柔,加极淡阴影。
- **AI 气泡**:透明 → 极淡白卡片 + 1px 暖描边,让 AI 回答成为"纸面"质感(书卷气),行高 1.72→1.78。
- AI 气泡内链接、加粗、列表的排版精修。

### 4. 空状态 `#examples-wrap`(视觉+布局)
- 欢迎语用衬线体 + 墨蓝,加一句更有温度的引导文案。
- 示例 chips:白底胶囊 → 加极淡墨蓝左边框/hover 时墨蓝描边+轻微上浮,交互反馈更明确。
- "换一换"按钮配色统一到墨蓝灰。

### 5. 悬浮输入区 `#input-row`(视觉+体验)
- 毛玻璃卡片保留,描边/阴影改暖调,聚焦时整条输入区加墨蓝光晕(`focus-within` 描边变墨蓝),交互态更清晰。
- 发送按钮:黑 → 墨蓝渐变,hover 加深 + 轻微放大。
- 清空按钮配色协调。

### 6. 深度模式开关 `#agent-mode-row`(视觉)
- checkbox 选中色改墨蓝,文案旁加一个小图标(🔍/⚡)提示,hover 有提示感。

### 7. 答案页脚 — 校验徽章 `.verify-badge`(视觉)
- ok 绿/warn 黄/none 灰三态保留语义,但描边和底色调淡、更精致,圆角统一。

### 8. 引用条文卡片 `.refs-block` / `.ref-item`(视觉+体验)
- 折叠块标题加墨蓝竖条标识(`border-left`),区分层级。
- 单条卡片 hover 墨蓝描边,展开箭头动画保留。
- 锚点高亮 `:target`:黄闪 → 墨蓝 wash 渐隐(与新配色一致),更协调。
- 条号锚点链接 `.cite-link`:虚线下划线 → 墨蓝实线 + hover 墨蓝 wash 底。

### 9. 多跳轨迹块 `_trace_block_html`(视觉)
- 复用引用块新样式,节点标签(规划/检索/反思/作答)加墨蓝小圆点,层次更清晰。

## 四、Python 侧改动(小幅)

`src/web.py` 内,**仅以下结构性微调**(不碰逻辑):
1. `CUSTOM_CSS` 常量整体重写(主要工作量)。
2. 顶栏 HTML:加分隔线元素 + 图标调整(`gr.HTML` 内容微调)。
3. 空状态欢迎语文案润色(`#examples-head` 的 HTML)。
4. 输入框 placeholder 文案可选微调。
5. 可选:`gr.themes.Soft()` 的主色种子设为墨蓝(`primary_hue`),让 Gradio 原生组件底色更协调 —— 需在 `app.py`/`main.py` 两处 `launch(theme=...)` 同步,或保持 Soft 不动只靠 CSS 覆盖(更稳,推荐后者)。

**不改**:`chat_stream`、流式逻辑、`on_like` 反馈、`_linkify_citations`、锚点机制、示例池、所有事件绑定。

## 六、风险与回滚

- **风险**:Gradio 版本对 `.message.user`/`.bubble-wrap` 等内部 class 有依赖,升级可能失效。当前 CSS 已大量用这些选择器,沿用同款即可,风险低。
- **回滚**:改动集中在一个常量 + 少量 HTML,`git checkout src/web.py` 即可完全还原。
- **不引入新依赖**,纯 CSS + 现有 Gradio 组件。
